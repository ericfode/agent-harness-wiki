# Labels and event processes in the asbestos operating system

**Source:** [DOI](https://doi.org/10.1145/1095809.1095813)
**Authors:** Petros Efstathopoulos, Maxwell Krohn, Steve VanDeBogart, Cliff Frey, David Ziegler, Eddie Kohler
**Date:** 2005

## Core idea
Asbestos combines label-based information-flow control with event processes so one system can safely handle work for multiple principals without leaking across compartments.

## Harness takeaway
The studio should support labeled memory compartments and per-task execution contexts even when many roles or users share one runtime.
